## 結論

**E1 書き込み機構は正常。書き込み漏れはゼロ。**
白書の仮説「E1が書き込みに失敗している回がある疑い」は否定された。

## 全数値の整合性

| 計測源 | 件数 | 意味 |
|---|---|---|
| paper_wallet SELL (リセット後) | 32件 | 物理的SELLイベント |
| ChromaDB trade_result (リセット後) | 32件 | ← E1完璧に記録 |
| ChromaDB trade_result (全期間) | 63件 | リセット前31 + リセット後32 |
| ラウンドトリップ | 25件 | 全量決済した回数 |
| Evaluator | 84件 | FIFOの BUY単位ペア分解数 |

## 混乱の原因

- **Evaluator 84**: BUY数ベースのFIFOペア分解（3BUY/1SELL=3ペア）
- **ChromaDB 63**: SELLイベントごとの記録（32件+31件のリセット前残存）
- → 両者は異なる単位を数えていた。比較に意味がなかった。
